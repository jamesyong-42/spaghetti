Navigation State Machine

How the browse command transitions between views

State Transitions
PROJECTS
List all projects
↑↓ navigate
Enter →
← Esc
SESSIONS
Sessions for project
↑↓ navigate
Enter →
← Esc
MESSAGES
Compact message list
↑↓ navigate
Enter →
← Esc
MESSAGE
Full message render
↑↓ scroll
Press q at any level → exit to normal terminal
Press Esc at PROJECTS level → also exits

State Details

PROJECTS
Header: Projects (5)
Items: Card rows (name, prompt, stats)
Enter: Push SESSIONS for selected project
Esc: Exit browser
SESSIONS
Header: project-name > Sessions (12)
Items: Card rows (number, prompt, stats)
Enter: Push MESSAGES for selected session
Esc: Pop back to PROJECTS (remembers position)
MESSAGES
Header: project > Session 2 > Messages (47)
Items: 2-line rows (role+time, preview)
Enter: Push MESSAGE detail view
Esc: Pop back to SESSIONS (remembers position)
MESSAGE (detail)
Header: project > Session 2 > Message 5
Content: Full renderMessage() output
↑↓: Scroll through content
Esc: Pop back to MESSAGES list
